LAMP 서버 구축

LAMP서버란 ?
Linux - Apache - MySql(MariaDB) - PHP 조합의 웹서버를 뜻함.
가장 대표적인 서버 조합.

apache는 기본적으로 PHP를 해석할 능력이 없음. 따라서 PHP와 MySql 등과 연동을 설치과정에서 설정해줘야함.
패키지 설치에서는 이 연동 설정이 꽤나 복잡하고 어려우므로 소스설치가 필수적임.

1. 웹서버(Apache)

Apache는 기본적으로 HTML만 해석 및 서비스 할수 있음.
php 문법을 해석할수 있는 모듈(라이브러리)를 추가적으로 아파치에 설치해야함

모듈의 설치방식
정적 모듈 방식
설치시부터 모듈을 포함한 상태로 설치하는 방식. 프로그램 부팅 속도는 느리겠지만 웹 호스팅 속도는 더 빠름, 호환성이 안좋음
모듈이 메모리에 상주하기 때문에 효율성에서 불리.
동적 모듈 방식 (Dynamic shared object) DSO
부팅시 메인 프로그램만 키면 되기 때문에 부팅속도가 빠름. 클라이언트의 요청이 있는 경우에만 모듈을 로딩하기 때문에 메모리 효율성이 좋음.
그러나 필요시 모듈을 로딩하는 시간이 걸려 호스팅 시간이 느려지게 됨.

But 어차피 Apache는 정적 모듈방식을 지원중단함. ㅎ..

멀티스레딩
mpm : multi processing module
기존에는 클라이언트의 요청 하나를 하나의 프로세스(단일스레드)가 담당하는 프로세스 기반 동작방식이었으나,
아파치 2.0으로 업데이트 되면서 하나의 프로세스가 다수의 쓰레드를 이용해 다수의 요청을 처리하는멀티스레드 방식을 지원하게 됨

prefork 타입 : 하나의 접속자를 하나의 프로세스가 동작하게 되는 단일스레드 방식 (기존방식)
worker 타입 : 하나의 프로세스(worker) 가 여러개의 쓰레드를 발생시켜 작동하는 방식, 전역변수나 모듈을 공통으로 사용할수 있게되어 램 호율성이 증가하였다.
event 타입 : keep_alived 방식을 적용, Request와 keep-Alive 한 요청을 Dedicated Thread 가 먼저 받아 분산처리를 진행한다.

apache 설치

C 기반 configure - make - make install 설치방식

1. 소스 다운로드

아파치 다운로드 링크

mv httpd-2.4.58.tar.bz2 /usr/local/src
tar jxf httpd-2.4.58.tar.bz2

추가 관련 패키지

  • XML parser
    dnf install expat expat-devel

  • pcre 설치
    pcre2-10.42.tar.bz2 다운로드

tar jxf pcre2-10.tar.bz2
cd pcre2-10.42
./configure --prefix=/usr/local/pcre
make
make install
  • apr / apr-util 모듈 준비
    apr-1.7.4.tar.bz2 , apr-util-1.6.3.tar.bz2 소스 다운로드
tar jxf apr-1.7.4.tar.bz2
tar jxf apr-util-1.6.3.tar.bz2

cp -a apr-1.7.4 /usr/local/src/httpd-2.4.58/srclib/apr 
cp -a apr-util-1.6.3 /usr/local/src/httpd-2.4.58/srclib/apr-util
  1. Configure
./configure —prefix=/usr/local/apache --enable-so --enable-mods-shared=all --with-included-apr --with-pcre=/usr/local/pcre 

--prefix => 기본 설치 디렉토리
--enable-so => 아파치의 동적컴파일 모듈을 활성화
--enable-mods-shared => 동적컴파일 레벨 ( all로 설정 )
--with-include-apr => apr 라이브러리를 포함하여 컴파일
--with-pcre=/usr/local/pcre =>  pcre 프로그램을 포함하여 컴파일

3. make

4. make install

5. 실행데몬스크립트
/usr/local/apache/bin/apachectl [start/stop/restart]

6. 환경설정파일
/usr/local/apache/conf/httpd.conf

주요 설정 옵션
- ServerName -> \www.cinnamon.com:80
- Document Root -> 웹문서 디렉토리 "/usr/local/apache/htdocs"
- Directory Index -> Index.html

★ 만약 이전버전의 httpd가 설치되어있다면 중지 및 삭제 해야한다.
rpm -e --nodeps httpd httpd-tools httpd-filesystem
★ 설치 및 환경설정 과정에서 옵션 오타와 같은 오류로 컴파일 중단될경우 해당 디렉토리를 삭제하고 처음부터 다시 설치해야한다
rm -rf httpd-2.4.58

2. Database (MySql)

DBMS (DataBase Manage System)
파일시스템의 한계를 해결하기 위해서 개발된 프로그램.
모든 페이지를 HTML등의 페이지로 저장하면 저장공간 효율성 및 부팅 및 로딩시간이 매우 오래걸림.
또한 일반적인 파일시스템은 다중접근이 불가함.

메인프레임과 기능만 로딩후 DB를 통해 정보만 로딩해 실행하는 방식.

Source Code -> 필요한파일(라이브러리,모듈) 이 전부 포함되어있음

3. PHP

"PHP: Hypertext Preprocessor"의 재귀적 약자로써 the php group 에서 개발 및 관리중이다.
대표적인 서버사이드 스크립트 언어로 C-like 문법으로 되어있고 자료구조가 간편하며 웹 관련 함수들이 많아 웹페이지 제작에 범용 스크립트 언어로 사용되고 있다

PHP 설치

1. 사전 필요 패키지

  • xml 라이브러리
    dnf install libxml2 libxml2-devel
  • sqlite3 라이브러리
    dnf install sqlite-devel

2. 소스 다운로드

PHP 다운로드 링크

cd /usr/local/src
tar jxf php-8.2.12.tar.xz

cd php-8.2.12

3. configure

./configure --with-apxs2=/usr/local/apache/bin/apxs --with-config-file-path=/usr/local/apache/conf 
--with-mysqli --with-pdo-mysql=/usr/local/mysql

--with-apxs2=  : PHP가 아파치 데몬에 DSO 모듈로써 로딩되도록 하는 실행명령어 `apxs`의 위치를 지정하는 옵션
--with-config-file-path=   : php.ini (초기설정파일) 이 들어갈 디렉토리 위치를 지정하는 명령어. 설치후 직접 넣어줘야 한다
--with-mysqli --with-pdo-mysql : 이전에는 mysql DB와 연결할때 mysqli 라는 방식을 사용했지만 보안관계상 PDO (PHP DATA OBJECT) 라는 방법을 권장하고있다.

**4. make

5.make install

**6. php.ini 파일 복사

cp php.ini-production /usr/local/apache/conf/php.ini

7. httpd.conf 파일 설정

아파치 웹서버가 php 확장자를 해석할수있도록 Addtype설정이 등록되는 mime_module 영역에 다음과 같이 추가해야한다.

vi /usr/local/apache/conf/httpd.conf

AddType application/x-httpd-php .php .htm .html .inc .php8  
AddType application/x-httpd-php-source .phps

8. 아파치 데몬 재시작

/usr/local/apache/bin/apachectl restart
- 서버네임 관련 경고메시지가 나타난다면 ip주소나 가상도메인을 설정하면 된다.
- 가상도메인 ->/etc/hosts 파일에 등록
