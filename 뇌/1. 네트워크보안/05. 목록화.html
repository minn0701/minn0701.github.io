  1. 목록화

목록화란 얻어낸 정보를 잘 정리하는것
실제 공격 실행의 바로 전단계

1. 풋프린팅

신문,게시판 등의 정보를 통해 사회공학적 기법으로 공격 대상에 대한 정보를 수집하는일.
비 기술적인 방식 인간에 대한 접근

웹사이트의 사용자 및 상담직원 문의메일주소 등의 사용자 계정을 알아내거나 부서의 팀장에 대한 개인정보를 수집하는일
게시판의 시스템 업그레이드 안내글, 현재 운영중인 서비스에 대한 안내등에 대한 글

2. 스캔

서버의 작동여부와 서버가 제공하는 서비스를 확인하는 작업

PING (ICMP echo request - reply)

시스템이 정상 작동중인지 확인하기 위한 간단한 툴

ping 192.168.56.105

운영체제에 따라 TTL 값과 패킷 갯수가 다르다.
리눅스는 255 윈도우는 64 우분투는 128

PORT 스캔

TCP Open Scan

TCP를 이용한 가장 기본적인 스캔
TCP 3-way handshake 연결이 완전히 성립되기 때문에 로그가 남는다는 단점이 있다.

포트가 열려있을 경우

공격자 ---> SYN ---> 공격대상
공격자 <--- SYN + ACK <--- 공격대상
공격자 ---> ACK ---> 공격대상

포트가 닫혀있을 경우

공격자 ---> SYN ---> 공격대상

공격자 <--- RST + ACK <--- 공격대상

Stealth scan

스텔스 스캔은 연결을 완전히 성립시키지 않고 공격 대상 시스템의 포트 활성화 여부에 대해서만 알아내는 방법
로그가 남지 않기 때문에 공격대상의 시스템 관리자는 공격 시도자의 IP를 알수 없다.

TCP Half Scan

공격자 ---> SYN ---> 공격대상
공격자 <--- SYN + ACK <--- 공격대상
공격자 ---> RTS ---> 공격대상

Fin / Xmas / Null Scan
열린 경우

공격자 ---> Fin/null/xmas ---> 공격대상
공격자 <--- 응답없음 <--- 공격대상

닫힌 경우

공격자 ---> Fin/null/xmas ---> 공격대상
공격자 <--- RST <--- 공격대상

UDP Scan

열린 경우

공격자 ---> UDP ---> 공격대상
공격자 <--- <--- 공격대상

닫힌 경우

공격자 ---> UDP ---> 공격대상
공격자 <--- ICMP unreachable <--- 공격대상

fping

네트워크 대역 전체에 확성화 시스템을 찾는 일을 스위핑(sweeping) 이라고 한다
이 스위핑을 가능하게 하는 자동화 툴이 fping이다

fping -q -a -s -g 192.168.15.0/24


-q : 통신에 성공한 호스트만 표시함
-a : 응답하는 호스트만 표시함
-g : 그룹지정, 대역에 포함된 호스트에게 전부 ping을 보냄
-s : 스캔이 끝난후 결과를 정리해서 보여줌

nmap

포트스캔을 위한 가장 강력한 툴

nmap -sT 192.168.15.202

-sT : TCP open scan
-sS : TCP stealth scan
-sF : TCP Fin scan
-sN : null scan
-sX : Xmas scan
-p  : 특정 포트 스캔
-f  : fragment , 방화벽을 통과하기 위해 패킷을 단편화 한다.

2. 운영체제 탐지

배너 그래빙

상대 시스템의 운영체제를 확인하는 기본적인 방법이다.
telnet ftp 등의 원격지 시스템에 로그인을 하면 나타나는 안내문 "배너" 를 확인한다. 

telnet 192.168.20.200 21

20.200 , 포트 21번에 대한 배너 그래빙

TCP/IP 반응을 이용한 탐지

TCP/IP 패킷에 대한 반응을 본다. TTL/패킷의 시퀀스 넘버등

넷크래프트

www.netcraft.com
입력한 사이트의 시스템리부팅 주기 시스템 갱신주기 현재 운영체제 버전 웹서비스데몬의 종류등을 보여줌

3. 방화벽 탐지

클라이언트가 접속을 시도하면 1차적으로 방화벽에서 허용과 차단을 결정하게 된다.

traceroute

TTL값을 1씩 늘려가면서 목적지까지의 네트워크상의 방화벽의 유무와 위치를 확인한다.

tracert www.google.com

1   <1ms  <1ms  192.168.15.2
2   <4ms  <4ms  192.168.1.1
3   *     *     요청시간이 만료되었습니다.  <= 방화벽 위치
4   <1s   <1ms  192.168.15.2

참고로 방화벽에 닫힌 포트에 보낸 패킷은 응답이 돌아오지 않으나, 열린포트에 보낸 패킷은 ICMP Time exceed 패킷이 돌아온다

SNMP

simple network manage protocol
TCP/IP 기반의 네트워크는 기본적으로 네트워크단에서 다양한 정보를 교환,제공한다. 그중 네트워크에 대해 관리하는 프로토콜이 snmp이다
snmp는 네트워크 장비의 통제를 위해 같은 통제범위에 있는 pc나 네트워크 장비에 이름을 부여한다.
PDU (product Data unit)타입을 지정할수 있고 DES와 MD5 를 이용한 보안을 지원한다.

Agent (관리대상)
- snmp (전송프로토콜 )
- mib ( 관리할 개체의 집합 )
- smi ( 관리방법 )

SMI (structure of management information)
관리 정보 구조 라고 한다.
