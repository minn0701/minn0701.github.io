서버 점검 스크립트 작성 실습

실습 과제

주요정보통신기반시설 기술적 취약분석 평가 방법 상세가이드 ( 이하 주통기 ) 의 W-01. W-02, W-04 W-06 항목을 점검하는 스크립트를 작성하라

W-01 Administrator 계정 이름 변경 또는 보안성 강화

1. 점검내용

윈도우즈 최상위 관리자 계정인 Administrator의 계정명 변경 또는 보안을 고려한 비밀번호 설정 여부 점검

2. 점검목적

윈도우즈 기본 관리자 계정인 Administrator의 이름을 변경 또는 보안을 고려한, 잘 알려진 계정을 통한 악의적인 패스워드 추측 공격을 차단하고자 함

3. 보안위협

일반적으로 관리자 계정으로 잘 알려진 Administrator를 변경하지 않은 경우 악의적인 사용자의 패스워드 추측 공격을 통해 사용 권한 상승의 위험이 있으며,
관리자를 유인하여 침입자의 액세스를 허용하는 악성코드를 실행할 우 려가 있음 윈도우즈 최상위 관리자 계정인 Administrator는 기본적으로 삭제하거나
잠글 수 없어 악의적인 사용자의 목표가 됨

W-02 Guest 계정 비활성화

1. 점검내용

Guest 계정 비활성화 여부 점검

2. 점검목적

Guest 계정을 비활성화 하여 불특정 다수의 임시적인 시스템 접근을 차단하 기 위함

3. 보안위협

Guest 계정은 시스템에 임시로 액세스해야 하는 사용자용 계정으로, 이 계 정을 사용하여 권한 없는 사용자가 시스템에 익명으로 액세스할 수 있으므로
비인가자 접근, 정보 유출 등 보안 위험이 따를 수 있음

W-04 계정 잠금 임계값 설정

1. 점검내용

계정 잠금 임계값의 설정 여부 점검

2. 점검목적

계정 잠금 임계값을 설정하여 공격자의 자유로운 자동화 암호 유추 공격을 차단하기 위함

3. 보안위협

공격자는 시스템의 계정 잠금 임계값이 설정되지 않은 경우 자동화된 방법 을 이용하여 모든 사용자 계정에 대해 암호조합
공격을 자유롭게 시도할 수 있으므로 사용자 계정 정보의 노출 위험이 있음

W-06 관리자 그룹에 최소한의 사용자 포함

1. 점검내용

관리자 그룹에 불필요한 사용자의 포함 여부 점검

2. 점검목적

관리자 그룹 구성원에 불필요한 사용자의 포함 여부를 점검하여, 관리 권한 사용자를 최소화 하고자 함

3. 보안위협

Administrators와 같은 관리자 그룹에 속한 구성원은 컴퓨터 시스템에 대한 완전하고 제한 없는 액세스 권한을 가지므로,
사용자를 관리자 그룹에 포함 시킬 경우 비인가 사용자에 대한 과도한 관리 권한이 부여될 수 있음

주통기 점검 스크립트 작성

echo ============W-01==============

$adminMembers = Get-LocalGroupMember -Group "Administrators"
$adminExists = $adminMembers | Where-Object { $_.Name -like "*Administrator*" }


if ($adminExists) {
    Write-Output "Administrator Account name is not changed"
   $w01 = 0

} else {
    Write-Output "Administrator Account name is changed"
	$w01 = 1
}

echo ============W-02==============

$guestAccount = Get-LocalUser -Name "Guest"

if (-not $guestAccount.Enabled -or $guestAccount.Enabled -eq $false) {
    Write-Output "Guest is not enabled."
   $w02 = 1 
} else {
    Write-Output "Guest is enabled."
   $w02 = 0
}

echo ============W-04==============

$users = Get-LocalUser
$lockoutThreshold = (secedit /export /cfg $env:TEMP\secpol.cfg) -match 'LockoutBadCount\s*=\s*(\d+)'
$lockoutThresholdValue = if ($lockoutThreshold) { [int]$matches[1] } else { 0 }

$accountsWithNoLockoutThreshold = @()


if ($lockoutThresholdValue -eq 0) {
    foreach ($user in $users) {
        $accountsWithNoLockoutThreshold += $user.Name
    }
}

$accountCount = $accountsWithNoLockoutThreshold.Count
$w04 = $accountCount
Write-Output "Number of accounts with no lockout threshold: $accountCount"
if ($accountCount -gt 0) {
    Write-Output "Accounts with no lockout threshold:"
    $accountsWithNoLockoutThreshold | ForEach-Object { Write-Output $_ }
}

Remove-Item $env:TEMP\secpol.cfg -ErrorAction SilentlyContinue


echo ============W-06==============

$adminMembers = Get-LocalGroupMember -Group "Administrators"

$adminNames = @()

foreach ($member in $adminMembers) {
    $adminNames += $member.Name
}

$memberCount = $adminNames.Count
$w06 = $memberCount
Write-Output "Number of Administrator Accounts: $memberCount"
if ($memberCount -gt 0) {
    Write-Output "Account Names:" $adminNames | ForEach-Object { Write-Output $_ }
}

echo "======= check result =========="
if ($w01 -eq 0) {
    Write-Output "W-01 is not safe"
} else {
    Write-Output "W-01 is safe"
}

if ($w02 -eq 0) {
    Write-Output "W-02 is not safe"
} else {
    Write-Output "W-02 is safe"
}

if ($w04 -eq 0) {
    Write-Output "W-04 is safe"
} else {
    Write-Output "W-04 is not safe"
}

if ($w06 -gt 2) {
    Write-Output "W-06 is not safe"
} else {
    Write-Output "W-06 is safe"
}

echo ============end==============


# Wait for a key press before exiting
echo "press any key to quit "
[System.Console]::ReadKey() | Out-Null

W-01

목적: 시스템의 "Administrators" 그룹에 있는 사용자 계정 중 이름에 "Administrator"가 포함된 계정이 있는지 확인합니다.
결과: "Administrator" 계정이 존재하면 $w01을 0으로 설정하고 "변경되지 않음" 메시지를 출력합니다.
존재하지 않으면 $w01을 1로 설정하고 "변경됨" 메시지를 출력합니다.

W-02

목적: "Guest" 계정이 활성화되어 있는지 확인합니다.
결과: 게스트 계정이 비활성화되어 있으면 $w02를 1로 설정하고 "비활성화됨" 메시지를 출력합니다.
활성화되어 있으면 $w02를 0으로 설정하고 "활성화됨" 메시지를 출력합니다.

W-04

목적: 계정 잠금 임계값이 설정되어 있는지 확인하고, 설정되어 있지 않은 계정을 나열합니다.
결과: 잠금 임계값이 0이면 모든 사용자 계정을 확인하여 잠금 임계값이 없는 계정을 $accountsWithNoLockoutThreshold 배열에 추가합니다.
잠금 임계값이 없는 계정의 수를 $w04에 저장하고, 해당 계정의 이름을 출력합니다.

W-06

목적: "Administrators" 그룹에 있는 계정의 수를 확인합니다.
결과: 관리자 계정의 수를 $w06에 저장하고, 계정 수를 출력합니다.
계정이 하나 이상이면 계정 이름도 출력합니다.

결과

목적: 각 변수의 값을 기반으로 시스템의 안전성을 평가합니다.
결과: 각 W-01, W-02, W-04, W-06에 대해 안전성 메시지를 출력합니다.

실행 결과 화면

실행한 윈도우 환경이 Windows 11 home 에디션이라 계정임계값에 대한 설정이 존재하지 않아 모든 계정에 대해 임계값을 찾을수 없다고 출력됨.

이렇게 해서 script.ps1 파일을 powershell 로 실행하는것 만으로 각 주통기 항목 4가지에 대해 한번에 점검할수 있는 스크립트를 작성해보았습니다.
